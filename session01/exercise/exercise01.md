# 第1回 実習手順書: 開発環境の準備とインターネットの仕組み

この手順書に沿って実習を進めてください。
各手順を完了したらチェックボックスにチェックを入れましょう。

> **スクリーンショットの撮り方**
> - **Windows**: `Windows + Shift + S`（範囲選択キャプチャ） または `Print Screen`（全画面キャプチャ）
> - **Mac**: `Cmd + Shift + 4`（範囲選択キャプチャ） または `Cmd + Shift + 3`（全画面キャプチャ）
> - **Chromebook**: `Ctrl + Shift + ウィンドウ切り替えキー`
> - スクリーンショットは自動的にデスクトップまたは「ピクチャ」フォルダに保存されます
> - 提出時にはファイル名を `実習番号_氏名` の形式にしてください（例: `実習1_山田太郎.png`）

## 実習1: 完成版TODOアプリを触ってみよう（10分）

### 目的
完成版のTODOアプリを操作し、Webアプリがブラウザとサーバー間で通信していることを確認します。

### 手順

- [ ] 1. ブラウザ（Chrome推奨）を開く
- [ ] 2. 講師から共有されたTODOアプリのURLにアクセスする
- [ ] 3. TODOアプリで以下の操作を行う:
  - [ ] タスクを3つ追加する（例:「買い物に行く」「レポートを書く」「部屋を掃除する」）
  - [ ] 1つのタスクを完了状態にする（チェックを付ける）
  - [ ] 1つのタスクを削除する
- [ ] 4. ブラウザの**開発者ツール**を開く
  - Windows: `F12` キー または `Ctrl + Shift + I`
  - Mac: `Cmd + Option + I`
- [ ] 5. 開発者ツールの上部にある **「Network」タブ** をクリック
- [ ] 6. Networkタブを開いた状態で、TODOアプリで新しいタスクを追加する
- [ ] 7. Networkタブに通信ログ（行）が増えることを確認する
- [ ] 8. 通信ログの行をクリックして、詳細情報を確認する

### 確認ポイント
- TODOアプリの操作（追加・完了・削除）ができたか
- 開発者ツールのNetworkタブで通信が発生していることを確認できたか

### 提出物
- [ ] **スクリーンショット**: 開発者ツールのNetworkタブに通信ログが表示されている画面

## 実習2: GitHubアカウントの作成とリポジトリのFork（10分）

### 目的
GitHubアカウントを作成し、テンプレートリポジトリを自分のアカウントにコピー（Fork）します。

### 手順

- [ ] 1. ブラウザで https://github.com にアクセスする
- [ ] 2. アカウントを作成する（既にある人はログイン）
  - 「Sign up」をクリック
  - メールアドレスを入力
  - パスワードを設定（15文字以上、または8文字以上で数字と小文字を含む）
  - ユーザー名を入力（英数字とハイフンのみ。後から変更は面倒なので慎重に）
  - メール認証を完了する
- [ ] 3. 講師から共有されたテンプレートリポジトリのURLにアクセスする
- [ ] 4. 画面右上の **「Fork」** ボタンをクリックする
  - 「Fork」ボタンが見つからない場合は、ログインしているか確認する
- [ ] 5. 「Create fork」画面で、設定はそのままで **「Create fork」** をクリック
- [ ] 6. ページが切り替わり、自分のアカウントにリポジトリがコピーされたことを確認する
  - URL が `https://github.com/自分のユーザー名/リポジトリ名` になっていればOK
  - ページ左上に `自分のユーザー名 / リポジトリ名` と表示されている

### 確認ポイント
- GitHubアカウントが作成できたか
- リポジトリがForkされ、自分のアカウントにコピーがあるか

### 提出物
- [ ] **スクリーンショット**: Forkしたリポジトリのページ（URLバーに自分のユーザー名が表示されている状態）

## 実習3: GitHub Codespacesの起動とhello.txtの作成（10分）

### 目的
クラウド開発環境であるCodespacesを起動し、ファイルの作成とターミナルの操作を体験します。

### 手順

- [ ] 1. Forkした自分のリポジトリをブラウザで開く
- [ ] 2. 緑色の **「<> Code」** ボタンをクリックする
- [ ] 3. ポップアップで **「Codespaces」** タブを選択する
- [ ] 4. **「Create codespace on main」** をクリックする
- [ ] 5. 読み込みが完了するまで待つ（1〜2分かかります）
  - VS Codeに似た画面が表示されればOK
  - 画面が表示されない場合はページをリロードしてみてください
- [ ] 6. 画面の構成を確認する
  - 左側: サイドバー（ファイル一覧）
  - 右側: エディタ（コードを書く場所）
  - 下側: ターミナル（コマンドを実行する場所）
- [ ] 7. 新しいファイルを作成する
  - サイドバーのファイル一覧の上にある **「新しいファイル」アイコン**（紙にプラスのマーク）をクリック
  - ファイル名に `hello.txt` と入力して Enter
  - エディタが開くので、以下の内容を入力:
    ```
    Hello, World!
    私の名前は○○です。
    今日からWebアプリケーション基礎を学びます。
    ```
  - `○○` には自分の名前を入れてください
- [ ] 8. ファイルを保存する
  - Windows: `Ctrl + S`
  - Mac: `Cmd + S`
  - ファイル名の横の丸い点（未保存マーク）が消えれば保存完了
- [ ] 9. ターミナルを開く（既に開いている場合はスキップ）
  - 画面上部のメニュー → **Terminal** → **New Terminal**
  - または `` Ctrl + ` ``（バッククォートキー）
- [ ] 10. ターミナルで以下のコマンドを実行する:
  ```bash
  cat hello.txt
  ```
  - 先ほど入力した内容が表示されればOK

### 確認ポイント
- Codespacesが起動したか
- hello.txt が作成できたか
- `cat hello.txt` でファイルの内容が表示されたか

### 提出物
- [ ] **スクリーンショット**: ターミナルで `cat hello.txt` を実行した結果が表示されている画面

## 実習4: ターミナルでインターネットの仕組みを体感する（10分）

### 目的
`ping` と `nslookup` コマンドを使い、インターネットの仕組み（IPアドレス、DNS）を体験します。

### 手順

- [ ] 1. Codespacesのターミナルが開いていることを確認する
- [ ] 2. `ping` コマンドでサーバーとの通信を確認する:
  ```bash
  ping -c 4 google.com
  ```
  - `-c 4` は「4回だけ送信する」という意味です
  - 結果が表示されるまで数秒待ちます
  - `time=` の後の数字がサーバーまでの往復時間（ミリ秒）です
- [ ] 3. 結果を確認する
  - `4 packets transmitted, 4 received` のように表示されればOK
  - `100% packet loss` と出た場合は、ネットワーク接続を確認してください
- [ ] 4. `nslookup` コマンドでDNSの仕組みを確認する:
  ```bash
  nslookup google.com
  ```
  - 「Address:」の後に表示されるのが google.com のIPアドレスです
- [ ] 5. 他のサイトでも試してみる:
  ```bash
  ping -c 4 github.com
  nslookup github.com
  ```
- [ ] 6. （余裕がある人向け）好きなWebサイトで試してみる:
  ```bash
  ping -c 4 好きなサイトのドメイン
  nslookup 好きなサイトのドメイン
  ```

### 確認ポイント
- `ping` でサーバーからの応答（time=...）が表示されたか
- `nslookup` でIPアドレスが表示されたか
- google.com と github.com でIPアドレスが異なることを確認できたか

### 提出物
- [ ] **スクリーンショット**: `ping -c 4 google.com` と `nslookup google.com` の実行結果が両方表示されている画面（1枚にまとまらない場合は2枚でOK）

## 実習5: 開発者ツールでリクエスト/レスポンスを観察する（10分）

### 目的
ブラウザの開発者ツールを使い、HTTPリクエストとレスポンスの実際の流れを観察します。

### 手順

- [ ] 1. ブラウザで新しいタブを開く
- [ ] 2. **開発者ツール**を開く
  - Windows: `F12` または `Ctrl + Shift + I`
  - Mac: `Cmd + Option + I`
- [ ] 3. 開発者ツールの **「Network」タブ** をクリック
- [ ] 4. アドレスバーに `https://www.google.com` と入力して Enter を押す
- [ ] 5. Networkタブに通信ログが大量に表示されることを確認する
  - たくさんの行が表示されている = 1ページを表示するのに多くの通信が必要
- [ ] 6. 一番上の行（最初のリクエスト）をクリックする
- [ ] 7. 右側に表示される詳細情報で以下を確認する:
  - **「Headers」タブ** を選択
  - **Request URL**: どのURLにリクエストを送ったか
  - **Request Method**: `GET`（ページを取得するリクエスト）
  - **Status Code**: `200`（成功）
- [ ] 8. 他の通信ログもクリックして、どんな情報がやりとりされているか観察する
  - 画像ファイル、CSSファイル、JavaScriptファイルなど、様々な種類がある
- [ ] 9. （余裕がある人向け）Networkタブの上にあるフィルターボタンで種類を絞り込んでみる
  - 「Doc」: HTMLドキュメント
  - 「CSS」: スタイルシート
  - 「JS」: JavaScript
  - 「Img」: 画像

### 確認ポイント
- Networkタブに通信ログが表示されたか
- リクエストのMethod（GET）とStatus Code（200）を確認できたか
- 1ページ表示に複数の通信が必要なことを理解できたか

### 提出物
- [ ] **スクリーンショット**: 開発者ツールのNetworkタブで、リクエストの詳細（Headers）が表示されている画面

## 実習6: 404エラーの確認と最初のGitコミット（10分）

### 目的
存在しないURLにアクセスして404エラーを確認し、Gitで最初のコミットを行います。

### 手順（Part 1: 404エラーの確認）

- [ ] 1. 開発者ツールのNetworkタブを開いた状態にする
- [ ] 2. ブラウザのアドレスバーに以下のURLを入力して Enter:
  ```
  https://github.com/thispagedoesnotexist12345
  ```
- [ ] 3. 「404」エラーページが表示されることを確認する
  - GitHubの「404 This is not the web page you are looking for」ページが表示される
- [ ] 4. 開発者ツールのNetworkタブで、ステータスコードが **404** になっていることを確認する
  - 一番上の行の「Status」列に `404` と表示されている

### 手順（Part 2: 最初のGitコミット）

- [ ] 5. Codespacesのターミナルに戻る
- [ ] 6. 現在のGitの状態を確認する:
  ```bash
  git status
  ```
  - `hello.txt` が `Untracked files`（未追跡ファイル）として表示されるはず
  - これは「Gitがまだ管理していない新しいファイルがある」という意味
- [ ] 7. hello.txt をステージング（コミット対象に追加）する:
  ```bash
  git add hello.txt
  ```
- [ ] 8. もう一度状態を確認する:
  ```bash
  git status
  ```
  - `hello.txt` が `Changes to be committed`（コミット予定の変更）に移動しているはず
- [ ] 9. コミットする（セーブポイントを作る）:
  ```bash
  git commit -m "hello.txtを作成"
  ```
  - `1 file changed` のようなメッセージが表示されればOK
- [ ] 10. コミット履歴を確認する:
  ```bash
  git log --oneline
  ```
  - 自分が作ったコミット `hello.txtを作成` が表示されるはず
  - 表示を終了するには `q` キーを押す

### 確認ポイント
- 404エラーページが表示されたか
- 開発者ツールで404ステータスコードを確認できたか
- `git commit` が成功したか
- `git log` で自分のコミットが表示されたか

### 提出物
- [ ] **スクリーンショット1**: 開発者ツールで404ステータスコードが表示されている画面
- [ ] **スクリーンショット2**: ターミナルで `git log --oneline` を実行した結果が表示されている画面

## 困ったときは

- **Codespacesが起動しない**: ページをリロードしてみてください。それでもダメな場合はブラウザを変えてみてください（Chrome推奨）。
- **コマンドが見つからないと言われる**: コマンドのスペルを確認してください。大文字/小文字は区別されます。
- **pingが終わらない**: `Ctrl + C` で停止できます。`-c 4` オプションを忘れていませんか？
- **git commitでエラー**: ユーザー名とメールアドレスの設定が必要な場合があります。以下を実行してください:
  ```bash
  git config user.name "自分の名前"
  git config user.email "自分のメールアドレス"
  ```
- **その他のトラブル**: 講師またはTAに質問してください。チャットでの質問もOKです。
